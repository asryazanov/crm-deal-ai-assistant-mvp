# Мини-ТЗ: AI-Ассистент в карточке сделки

## Цель MVP

Сделать локальный демонстрационный web-MVP, который на тестовом датасете сделок показывает:

- вероятность выигрыша сделки по прозрачной rule-based методологии;
- здоровье сделки: зеленая, желтая или красная зона;
- ключевые риски;
- рекомендации отдельно для менеджера и руководителя.

MVP работает локально, без внешних AI API и без отправки данных за пределы рабочего места.

## Источники данных

Для MVP используется файл `Axoft_Deal_Dataset_v1.0.xlsx`, преобразованный в `data/deals.json`.

В целевом контуре источники:

- CRM: карточка сделки, активности, контакты, команда сделки;
- 1С: финансовые и складские данные;
- история продаж: клиент, партнер, связка партнер-клиент.

## Пользовательский сценарий

1. Пользователь открывает web-страницу ассистента.
2. Видит список сделок и сводные показатели по датасету.
3. Выбирает сделку или ищет ее по ID, клиенту, партнеру, менеджеру.
4. Ассистент пересчитывает вероятность по методологии.
5. В карточке отображаются:
   - вероятность закрытия;
   - здоровье сделки;
   - приоритет;
   - основные факторы роста и падения оценки;
   - риски;
   - план действий для менеджера;
   - план действий для руководителя.

## Формула скоринга

Скоринг начинается с базовой вероятности 50%. Далее применяются прозрачные поправки. Итог ограничивается диапазоном 5-95%.

## Здоровье сделки

Здоровье сделки рассчитывается отдельно от вероятности:

- Зеленая: вероятность >= 70%, критических рисков нет, высоких рисков не больше одного.
- Желтая: вероятность 45-69% или есть 1 критический риск / 2 высоких риска.
- Красная: вероятность < 45%, либо 2+ критических риска, либо 3+ высоких риска.

## Риски MVP

Критические:

- нет ЛПР;
- бюджет не подтвержден;
- коммуникаций нет более 21 дня.

Высокие:

- коммуникаций нет более 14 дней;
- нет BDM;
- нет инженера / presale;
- нет регистрации у вендора;
- слабый win rate партнера;
- слабая история связки партнер-клиент;
- сделка старше 180 дней.

Средние:

- нет пилота;
- нет заявки на пресейл;
- давно не было встречи с партнером;
- не заполнен контакт партнера;
- роль партнера сведена к цене.

## Ограничения MVP

- Используется rule-based модель, без ML-калибровки.
- Рекомендации шаблонные и формируются по найденным рискам.
- Нет записи задач обратно в CRM.
- Нет авторизации и ролевой модели.
- Нет интеграции с MS Dynamics, 1С и Power BI.
